什麼是構建？

現代前端開發的挑戰

隨著 Web 應用程式變得越來越複雜、更具規模，為了確保開發效率、程式碼品質、可維護性，開發者在原始碼中導入了許多提升開發體驗的功能和語法：

  • 熱模組替換 (HMR)：即時看到修改後的效果，無需手動重新整理頁面
  • TypeScript：提供靜態型別檢查，編譯時期就能捕捉到型別錯誤
  • JSX 語法：React 的 JSX 讓我們能用類似 HTML 的語法來描述 UI 結構，既直觀又易於理解
  • CSS 預處理器：如 Sass、Less 提供變數、嵌套、函數等功能
  • ES6 語法：使用最新的 JavaScript 語法特性來提升程式碼可讀性

然而，這些提升開發體驗的功能在生產環境中往往是非必要的，甚至無法直接在瀏覽器中運行。

原始碼 vs 生產環境的差異

原始碼階段 - 注重開發者體驗

  • 使用最新的語法和功能
  • 保持良好的程式碼結構和可讀性
  • 包含開發工具和除錯資訊
  • 模組化開發，檔案易於重複引用、管理

生產環境 - 注重使用者體驗

  • 追求最小的檔案大小
  • 追求最快的載入速度

雖然上述提到了開發、正式階段，不過更加精確來說，構建其實是指「生成正式環境程式碼的整個流程」。如果已經有使用框架的經驗，其實最常用到的「npm run build」背後就是去調用構建工具來完成這個流程。

構建的流程

構建的流程，涵蓋多種問題與解決方案，並且是逐步演化而來的，大致如下：

  • 開發體驗階段

    • 問題：修改程式碼後需手動刷新，開發效率低
    • 解決方案：Live Reload → Hot Module Replacement (HMR)

  • 編譯階段

    • 問題：瀏覽器不支援最新 JavaScript 語法、TypeScript、JSX
    • 解決方案：Babel（JS 轉譯）、TypeScript Compiler（型別檢查）

  • 資源處理階段

    • 問題：CSS、圖片、字型等靜態資源需要優化和管理
    • 解決方案：CSS 預處理器（Sass/Less）、圖片壓縮、字型處理

  • 效能優化階段

    • 問題：檔案過大、載入速度慢、快取策略
    • 解決方案：程式碼分割(Code Spliting)、Tree Shaking

既然有不同的問題與解決方案，那自然會有很多工具，比如：Webpack、Babel、SWC …，有些工具相當輕量只針對單個問題，有些則滿足多種需求。
而 Vite 正是多種構建工具之一，屬於可以滿足多種需求的類型。


Vite 登場

Vite 做為 Web 應用的構建工具，提升了開發體驗與應用性能。

開發體驗

  1. 提升開發伺服器的啟動速度

    在開發環境，將程式碼區分依賴與原始碼，並搭配現代瀏覽器接收 ESM（模組） 的機制，優化啟動速度。

    • 現代瀏覽器與 ESM

      允許解析 ESM 的 JavaScript 檔案（import / export），改善開發伺服器必須打包才能給瀏覽器的狀況。

    • 依賴

      指第三方套件（node_modules），在專案中套件程式碼的變動性低，因此 Vite 在啟動開發伺服器時會進行 pre-build，pre-build 的結果會快取（node_modules/.vite），而且是 ESM 格式。（ pre-build 透過 esbuild 工具實現打包 ）

    • 原始碼

      指開發者的原始碼，Vite 實現先啟動開發伺服器，再針對請求路由回傳 ESM 檔案給瀏覽器解析的機制，緩解傳統上必須先將應用程式打包( Bundle based dev server ) 才能啟動伺服器的狀況。
